Eléments de correction

La difficulté n'est pas dans la définition des éléments type livret ou autres (il y a plétore de types à réutiliser pour y arriver) mais dans la façon d'accéder aux types proposés par l'INSEE. On peut distinguer 2 possibilités :
  • Un schéma définissant le même espace de noms "http://xml.insee.fr/schema" pour lequel on peut utiliser des include. Voir le fichier livret-1.xsd

  • Un schéma définissant un autre espace de noms afin d'utiliser un import. Comme on ne peut pas faire plusieurs import différents pour un même espace de noms, cela nécessite de passer par un schéma intermédiaire (insee.xsd) pour définir le schéma principal livret-2.xsd
